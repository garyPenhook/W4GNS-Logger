═════════════════════════════════════════════════════════════════════════════

TWO STANDARD SKCC FIELDS:
────────────────────────

1. SKCC Field (Contacted Station)
   ├─ Field Name: <SKCC>
   ├─ Data Type: String (S)
   ├─ Contains: SKCC member number of CONTACTED station
   ├─ Format: <SKCC:5>12345C
   └─ Example: <SKCC:5>12345C

2. MY_SKCC Field (Your Station)
   ├─ Field Name: <MY_SKCC>
   ├─ Data Type: String (S)
   ├─ Contains: YOUR OWN SKCC member number
   ├─ Format: <MY_SKCC:6>14276T
   └─ Example: <MY_SKCC:6>14276T

═════════════════════════════════════════════════════════════════════════════

SKCC MEMBER NUMBER FORMAT:
──────────────────────────

Structure: [NUMBER][SUFFIX]

Suffixes (Achievement Levels):
  C    = Centurion        (100 SKCC member contacts)
  T    = Tribune          (50 additional C/T/S contacts)
  Tx1  = Tribune Level 1  (showing progression)
  Tx2  = Tribune Level 2
  ...
  Tx8  = Tribune Level 8  (max before Senator)
  S    = Senator          (highest level, 600+ total contacts)

Examples:
  12345C       = Centurion member #12345
  14276T       = Tribune member #14276
  14276Tx4     = Tribune member #14276 at level 4
  9876S        = Senator member #9876
  (blank)      = Non-SKCC member

ADIF QSO RECORD EXAMPLE:
───────────────────────

<CALL:5>W5XYZ
<QSO_DATE:8>20241021
<TIME_ON:4>1430
<BAND:3>40M
<MODE:2>CW
<FREQ:8>7122.500
<RST_RCVD:3>559
<RST_SENT:3>569
<SKCC:5>12345C              ← Contacted station SKCC number
<MY_SKCC:6>14276T           ← Your SKCC number
<COUNTRY:14>United States
<STATE:2>TX
<EOR>

═════════════════════════════════════════════════════════════════════════════

FIELD PLACEMENT IN ADIF:
────────────────────────

Standard order (recommended):
  1. <CALL> - Callsign
  2. <QSO_DATE> - Date
  3. <TIME_ON> - Time
  4. <BAND> - Band
  5. <MODE> - Mode (CW for SKCC)
  6. <FREQ> - Frequency
  7. <RST_RCVD> - RST received
  8. <RST_SENT> - RST sent
  9. <SKCC> - ← SKCC number here
 10. <MY_SKCC> - ← Your SKCC here
 11. Other fields (COUNTRY, STATE, etc.)
 12. <EOR> - End of record

═════════════════════════════════════════════════════════════════════════════

COMMON MISTAKES:
────────────────

❌ WRONG: Using comments field
   <COMMENT:18>SKCC member 12345C

✅ CORRECT: Using standard SKCC field
   <SKCC:5>12345C

─────────────────────────────────────

❌ WRONG: Stripping suffix
   <SKCC:5>12345

✅ CORRECT: Including suffix
   <SKCC:5>12345C

─────────────────────────────────────

❌ WRONG: Storing as integer
   <SKCC:5>12345

✅ CORRECT: Storing as string with suffix
   <SKCC:7>14276Tx4

─────────────────────────────────────

❌ WRONG: Not setting MY_SKCC
   (omitting your own SKCC)

✅ CORRECT: Including your SKCC
   <MY_SKCC:6>14276T

═════════════════════════════════════════════════════════════════════════════

HANDLING NON-MEMBERS:
─────────────────────

For stations that are NOT SKCC members:
  - Leave <SKCC> field blank
  - Or omit <SKCC> field entirely

Example:
<CALL:5>N0NEM
<QSO_DATE:8>20241021
<TIME_ON:4>1430
<BAND:3>40M
<MODE:2>CW
<MY_SKCC:6>14276T
<EOR>

IMPORTING/EXPORTING:
────────────────────

When importing ADIF files:
  ✓ Extract <SKCC> field value
  ✓ Store WITH suffix (C, T, S, Tx#)
  ✓ Don't strip or parse suffix
  ✓ Preserve as string

When exporting to ADIF:
  ✓ Include <SKCC> for members
  ✓ Include <MY_SKCC> always
  ✓ Preserve suffixes exactly
  ✓ Use String data type
  ✓ Omit for non-members

KEY TAKEAWAY:
─────────────

The SKCC number is NOT a custom field or comment.
It uses standard ADIF fields that have been officially
part of the specification since ADIF 3.0.3.

Use:      <SKCC> for contacted station's SKCC number
Use:      <MY_SKCC> for your SKCC number
NOT:      Comments, custom fields, or other workarounds
